# 30-Day React JS Practice Plan

This 30-day practice plan helps learners build React skills step by step, from core components to full frontend integration.

## Plan Overview

- Duration: 30 days
- Focus: React fundamentals, hooks, routing, state, and frontend patterns
- Structure: 5 weeks with one review day each week
- Outcome: a complete React application with data fetching, forms, and navigation

## Week 1: React Basics

### Day 1: React Setup and JSX
- Learn how to create a React app with Create React App or Vite.
- Practice writing JSX and rendering elements.
- Deliverable: a starter React app with components.

### Day 2: Components and Props
- Learn functional components and props.
- Practice building reusable UI components.
- Deliverable: a small component library with props.

### Day 3: State with `useState`
- Learn state handling in React.
- Practice building interactive components.
- Deliverable: a counter or toggle component.

### Day 4: Event Handling
- Learn React event syntax and callbacks.
- Practice form controls and button actions.
- Deliverable: a form that updates state on input.

### Day 5: Conditional Rendering
- Learn rendering based on state.
- Practice showing and hiding components.
- Deliverable: a UI that toggles content sections.

### Day 6: Review and Mini Challenge
- Review core React concepts.
- Practice building a small interactive widget.
- Deliverable: a mini React app using props, state, and events.

## Week 2: Component Patterns

### Day 7: Lists and Keys
- Learn rendering lists and using keys.
- Practice mapping data arrays to UI.
- Deliverable: a dynamic list component.

### Day 8: Forms and Controlled Components
- Learn controlled form inputs.
- Practice building a data entry form.
- Deliverable: a form with validation and state handling.

### Day 9: `useEffect` Basics
- Learn side effects and lifecycle behavior.
- Practice fetching data on mount.
- Deliverable: a component that loads API data.

### Day 10: Component Composition
- Learn component nesting and reuse.
- Practice creating a dashboard layout.
- Deliverable: a page built from reusable child components.

### Day 11: Context API
- Learn global state sharing with Context.
- Practice passing data without prop drilling.
- Deliverable: a theme or auth context provider.

### Day 12: Review and Mini Challenge
- Review intermediate React patterns.
- Practice building a small CRUD interface.
- Deliverable: a multi-component page with user interaction.

## Week 3: Routing and Data

### Day 13: React Router Basics
- Learn client-side routes and navigation.
- Practice adding multiple pages.
- Deliverable: a React app with routed views.

### Day 14: Dynamic Routes
- Learn route parameters and data-driven pages.
- Practice rendering detail pages.
- Deliverable: a list/detail page flow.

### Day 15: Data Fetching and APIs
- Learn fetching data from REST APIs.
- Practice error handling and loading states.
- Deliverable: a data-driven page with fetch logic.

### Day 16: Component Performance
- Learn memoization with `React.memo` and `useMemo`.
- Practice optimizing slow components.
- Deliverable: a performance-improved list or form.

### Day 17: Custom Hooks
- Learn creating reusable hooks.
- Practice abstracting logic from components.
- Deliverable: a custom hook for API fetching or form handling.

### Day 18: Review and Mini Challenge
- Review routing, data fetching, and hooks.
- Practice building a full feature page.
- Deliverable: a component with routing, data, and state.

## Week 4: UI and State Management

### Day 19: Styling React Apps
- Learn CSS modules, styled-components, or CSS-in-JS.
- Practice styling components responsively.
- Deliverable: a polished UI page.

### Day 20: State Management Patterns
- Learn lifting state, context, and local state.
- Practice managing shared data.
- Deliverable: a stateful feature using context or prop drilling.

### Day 21: Forms with Validation
- Learn form validation strategies.
- Practice showing validation messages.
- Deliverable: a validated form component.

### Day 22: Error Boundaries
- Learn error handling in React.
- Practice creating an error boundary component.
- Deliverable: a safe React UI with fallback rendering.

### Day 23: Animations and Transitions
- Learn simple UI animations.
- Practice animating component entry/exit.
- Deliverable: animated menu or list transitions.

### Day 24: Review and Mini Challenge
- Review advanced UI and state concepts.
- Practice building a complete frontend page.
- Deliverable: an interactive and polished UI component set.

## Week 5: Final React Project

### Day 25: App Planning
- Plan a small React project with multiple pages.
- Identify components, routes, and data flows.
- Deliverable: a project plan and component map.

### Day 26: Build Core Pages
- Build main app pages and layout.
- Practice navigation and shared components.
- Deliverable: app skeleton with nav and page structure.

### Day 27: Fetch and Display Data
- Connect the app to static data or an API.
- Practice rendering dynamic content.
- Deliverable: pages showing API or mock data.

### Day 28: Add Forms and Interactivity
- Add forms, filters, or search.
- Practice updating UI with user input.
- Deliverable: interactive app feature.

### Day 29: Polish and Fix Bugs
- Review the app, fix bugs, and improve UX.
- Practice cleanup and refactoring.
- Deliverable: a refined React app.

### Day 30: Review and Share
- Review the project and document what you built.
- Share or deploy the app if desired.
- Deliverable: a final project summary.

## Tips for Success

- Commit progress daily.
- Focus on understanding components and data flow.
- Build while learning and apply concepts immediately.
- Keep the app small but complete.
- Revisit code and improve it each week.
