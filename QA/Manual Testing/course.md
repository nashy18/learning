# Manual Testing Foundations for Beginners

## 1. Learning Objectives

- What the learner will achieve:
  - Understand the role of manual testing and how to execute effective test cases.
  - Learn constructing test plans, test scenarios, and bug reports.
  - Practice exploratory testing and usability validation.
- Skills gained:
  - Writing clear test cases, running manual execution, and logging defects.
  - Verifying functional behavior, UI flows, and edge cases.
- Real-world applicability:
  - Validate web and mobile applications manually before automation.
- Prerequisites:
  - Basic software literacy and understanding of application workflows.
- Common misconceptions:
  - Manual testing is not unstructured; it requires planning and precision.

## 2. Fundamental Concepts

### 2.1 Test Cases and Test Scenarios
- Definition: test cases are step-by-step checks; scenarios are end-to-end use cases.
- Why it exists: to ensure consistent coverage.
- Where it is used: QA executions, regression checks.
- How it works: describe preconditions, steps, expected results.
- Analogy: a test case is a recipe, a scenario is a full meal plan.
- Best practices: keep steps clear and concise.
- Common mistakes: missing expected results, assuming behavior.

### 2.2 Exploratory Testing
- Definition: ad hoc testing guided by experience.
- Why it exists: to discover issues beyond scripted checks.
- Where it is used: early reviews, complex flows.
- How it works: explore while observing behavior and learning the app.
- Analogy: detective work.
- Best practices: take notes and focus on risk areas.
- Common mistakes: no documentation.

### 2.3 Usability Testing
- Definition: assess whether the app is easy to use.
- Why it exists: to improve user experience.
- Where it is used: UI flows, sign-up, checkout.
- How it works: follow user tasks and note friction.
- Analogy: watching someone use a new appliance.
- Best practices: test with realistic tasks.
- Common mistakes: testing only with known users.

### 2.4 Defect Reporting
- Definition: documenting found issues.
- Why it exists: to communicate problems clearly.
- Where it is used: bug trackers and QA reports.
- How it works: record steps, expected and actual results.
- Analogy: filing a maintenance request.
- Best practices: include screenshots and severity.

### 2.5 Regression Testing
- Definition: verifying existing functionality after changes.
- Why it exists: to catch unintended breakages.
- Where it is used: after updates, fixes, and releases.
- How it works: re-run test cases for impacted areas.
- Analogy: quality checks after a repair.
- Best practices: prioritize high-risk flows.

## 3. Progressive Learning

### Level 1 – Basic Understanding
- Learn how to write a simple test case.
- Example: validate a login form.

### Level 2 – Intermediate Usage
- Create a test plan for a small module.
- Example: test a registration flow.

### Level 3 – Advanced Concepts
- Perform exploratory and usability testing.
- Example: test a shopping cart from a user perspective.

### Level 4 – Expert-level Implementation
- Combine manual and automated regression strategies.
- Example: manage a release checklist for a web app.
